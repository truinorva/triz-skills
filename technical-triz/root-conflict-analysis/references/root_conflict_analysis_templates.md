# RCA+ Templates, Solutions, and Report Structure

Reusable wording and reporting guidance for the Root Conflict Analysis (RCA+) skill.

---

## Lookup Map

- For wording contradictions and problems, use section 1.
- For solution directions after RCA+, use section 2.

## 1. Contradiction Formulation Templates

### Technical Contradiction (TC)

A technical contradiction pairs two contradicting effects of the same cause:

- "How to ensure [N+P-cause] to enable [positive effect] but to avoid [negative effect]?"
- "[Cause] leads to [positive effect], but also leads to [negative effect]."

### Physical Contradiction (PC)

A physical contradiction states opposite required states of the same parameter:

- "[N+P-cause] should be present/high/strong to [positive effect] AND should be absent/low/weak to avoid [negative effect]."
- "The system needs more [cause/parameter] for [useful result], but less [cause/parameter] for [harm reduction]."

### General Conflict Statement

- "[Design decision / cause] creates [positive effect], but also causes or increases [negative effect]."
- "The conflict is not merely that [negative effect] exists; the conflict is that the same underlying cause also provides [positive effect]."

### Problem Statement for N-type Causes

- "How to eliminate / prevent [N-cause]?"
- "How to reduce / control [N-cause parameter]?"

## 2. Solution Guidance

Do not jump to solutions before contradictions have been clearly stated and problems have been recommended (Steps 8 and 9).

### For N-type causes (purely negative)

Suggest direct actions:

- elimination
- mitigation
- relocation
- parameter adjustment
- process correction
- better measurement
- control action
- change of operating condition

### For N+P-type causes (contradictions)

Suggest inventive directions based on TRIZ separation principles and other methods:

- separation in time
- separation in space
- separation by condition
- separation by scale
- separation by system level
- dynamic adaptation
- segmentation
- feedback control
- selective activation
- local quality
- transition to a higher system level
- replacement of the harmful mechanism while preserving the useful effect

For every solution direction, explicitly explain how it preserves the positive effect while reducing the negative effect.

### Recommended TRIZ solving methods

After RCA+ identifies contradictions, the following TRIZ methods can be applied:

- Contradiction Matrix and 40 Inventive Principles (for technical contradictions)
- Separation Principles (for physical contradictions)
- 76 Standard Solutions and Substance-Field Analysis
- ARIZ (Algorithm of Inventive Problem Solving)
- Function-Oriented Search
- Feature Transfer

## 3. Final RCA+ Report Structure

When the user asks for a final report, use this structure:

1. Executive Summary
2. System Description and Improvement Objective
3. General Negative Effect and Problem Category
4. RCA+ Diagram (top-down cause-effect structure with AND/OR relationships)
5. Summary Table (ID | Cause | Type | Positive Effect | Negative Effect)
6. Problem Formulations (TC and PC for each N+P cause; elimination problems for N causes)
7. Recommended Problems for Solving (prioritized list with rationale)
8. Assumptions, Open Questions, and Verification Needs
9. Suggested Next Steps (which TRIZ methods to apply)
